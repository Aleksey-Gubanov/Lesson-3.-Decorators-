## 1️⃣ Задача 1: Базовый декоратор `logger`

**Цель:** Создать декоратор, который логирует имя функции, переданные аргументы, результат выполнения и временную метку в жестко заданный файл `main.log`.

### Архитектура решения
Используется классическая структура декоратора с одним уровнем вложенности (замыкание). 

**Ключевые особенности реализации:**
* **Сохранение метаданных:** Применен `@wraps` из `functools`. Это критически важно, чтобы `__name__` и `__doc__` декорированной функции не заменялись на имена внутренней функции-обертки.
* **Timestamp на каждый вызов:** `datetime.now()` генерируется внутри wrapper-функции, чтобы каждая запись в логе имела актуальную метку времени.
* **Безопасная работа с файлами:** Использование контекстного менеджера `with open(...)`. Это гарантирует закрытие файлового дескриптора даже в случае сбоя.
* **Режим добавления (`'a'`):** Файл открывается в режиме `append`, чтобы последующие вызовы не перезаписывали предыдущие логи.

## 2️⃣ Задача 2: Параметризованный декоратор `logger(path)`

**Цель:** Доработать декоратор так, чтобы путь к лог-файлу передавался как аргумент. Добавить логирование времени вызова (timestamp) и корректную обработку возникающих исключений.

### Архитектура решения
Параметризованный декоратор требует **трех уровней вложенности** (фабрика декораторов):
1. `logger(path)` — принимает аргументы и возвращает сам декоратор.
2. `__logger(old_function)` — принимает декорируемую функцию.
3. `new_function(*args, **kwargs)` — обертка, выполняемая при вызове.

**Ключевые особенности реализации:**
* **Обработка исключений (Try/Except/Raise):** Декоратор перехватывает `Exception`, записывает тип и текст ошибки в лог, а затем **пробрасывает её дальше** (`raise`). *Правило сеньора: декоратор не должен "глотать" ошибки, если это не предусмотрено бизнес-логикой.*
* **Timestamp:** Использование `datetime.now().strftime()` для добавления человекочитаемых меток времени, что необходимо для анализа инцидентов.
* **DRY (Don't Repeat Yourself):** Строка лога формируется в памяти, а запись в файл происходит один раз в конце блока `try/except`, чтобы минимизировать операции I/O.

## 3️⃣ Задача 3: Применение декоратора к ООП-коду

**Цель:** Применить написанный параметризованный декоратор к реальному коду из предыдущего домашнего задания (ООП-задача про студентов, лекторов и экспертов), чтобы продемонстрировать работу декоратора в контексте классов и методов.

### Архитектура решения
Взята ООП-система из предыдущего ДЗ, реализующая:
- Класс `Student` (студент с оценками за ДЗ)
- Класс `Lecturer` (лектор с оценками за лекции)
- Класс `Reviewer` (эксперт, проверяющий ДЗ)
- Механизмы выставления оценок, подсчета средних, сравнения объектов

**Декоратор применен к ключевым бизнес-методам:**
- `__init__` — логирование создания объектов
- `rate_hw`, `rate_lecture` — логирование выставления оценок
- `get_average_grade` — логирование подсчета средних оценок
- `average_grade_by_course` (статический метод) — логирование агрегации данных по курсам

**Ключевые особенности реализации:**
* **Порядок декораторов для `@staticmethod`:** Декоратор `@logger` должен стоять НИЖЕ `@staticmethod`, иначе получим ошибку (logger попытается декорировать объект `staticmethod`, а не функцию).
* **Выборочное декорирование:** Не все методы задекорированы — только те, что представляют бизнес-ценность для логирования. Магические методы (`__str__`, `__lt__`, `__gt__`) не декорируются, чтобы не засорять лог.
* **`self` в логах:** При декорировании методов экземпляров, `self` попадает в `args` и отображается в логе как `<__main__.Student object at 0x...>`. Это нормальное поведение.

### Пример записи в логе:
```
[2026-06-26 15:42:18] Вызов функции: rate_hw
[2026-06-26 15:42:18] Аргументы: args=(<__main__.Reviewer object at 0x7f8b1c>, <__main__.Student object at 0x7f8b2a>, 'Python', 10), kwargs={}
[2026-06-26 15:42:18] Возвращаемое значение: None
```

```bash
python task_3.py
```

*Результат:*
- В консоли отобразятся результаты работы ООП-системы (создание объектов, выставление оценок, подсчет средних)
- Создастся файл `task_3.log` с детальными записями о каждом вызове задекорированных методов
